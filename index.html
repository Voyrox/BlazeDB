How it works

A simple write path you can explain on a whiteboard: append to WAL, stage in memory, flush to sorted tables, and compact in the background.

1 Write path
Append to WAL

Durability starts with write ahead logging. Fsync policy is configurable for your risk/throughput needs.

2 In memory
Update memtable

In memory staging keeps the common path fast and predictable while batching work efficiently.

3 On disk
Flush to SSTables

Sorted immutable tables make reads and scans efficient and keep file formats straightforward.

4 Background
Compact

Compaction keeps space under control and maintains read performance without requiring constant tuning.

Get started

Run it locally in minutes. Use the docs for full deployment and configuration details.

Run the server

# Build + run (Ninja/CMake)
$ cmake -S . -B build -G Ninja
$ ninja -C build
$ ninja -C build run

# System install (Linux, systemd)
$ sudo ./install.sh

Connect

# Node.js driver
$ npm i xeondb-driver
$ node packages/test_nodeJS.js

# Go driver
$ go get github.com/Voyrox/Xeondb/packages/golang@latest
$ go run packages/test_golang.go

Docker

# Quick eval (container)
$ docker build -t xeondb .
$ docker run --rm -p 9876:9876 xeondb
